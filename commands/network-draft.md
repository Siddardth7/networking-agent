# /network-draft

Generate parallel outreach drafts for all selected contacts at a company, or a specific subset.

## Usage

```
/network-draft <company-slug>
/network-draft <company-slug> --contacts 1,2,4
/network-draft <company-slug> --revise <draft-id> --feedback "<feedback text>"
```

**Examples:**
```
/network-draft lockheed-martin
/network-draft spacex --contacts 2,3
/network-draft blue-origin --revise 42 --feedback "Too formal — make it more peer-to-peer"
```

## What Happens (default)

For each selected contact at the company, runs 3 LLM calls in parallel:

| Channel | Constraint |
|---|---|
| `LINKEDIN_CONNECTION` | ≤ 300 characters — connection request note |
| `LINKEDIN_POST_CONNECTION` | No limit — post-acceptance follow-up |
| `COLD_EMAIL` | ≤ 150 words — with subject line |

Each draft is personalized using:
- **Persona template** (`src/templates/personas/<persona>.md`)
- **Voice doc** (`~/.networking-agent/voice.md`, if present)
- **Matched achievements** (top 3 bullets from `~/.networking-agent/resume_library.yaml` by focus area + title keyword overlap)
- **Hook** (generated by Finder — UIUC / shared employer / title specialty / GENERIC)

## Reputation Guardrails

Every draft is checked against a blocklist of cringe phrases:
- `I noticed`, `I admire`, `I came across your company`, `your impressive work`

**On match:** regen once with explicit anti-phrase nudge.  
**On second match:** save draft with `quality_flag=true` — surfaced as ⚠️ in the Marketer review.

## Flags

`--contacts 1,2,4` — draft only for these contact numbers (from the last selection gate output).

`--revise <draft-id> --feedback "<text>"` — regenerate one specific draft with feedback.  
The Marketer's `REVISE` verb uses the dispatch protocol in Phase 7 (`src/agents/dispatch.py`) instead.

## Prerequisites

Run `/network-find <company-slug>` first and complete the selection gate.  
Contacts must be in `SELECTED` state.

## State Machine

```
SELECTED  →  DRAFTED  (after all 3 channels persisted per contact)
```
